# xzawed Suite 개요

xzawed는 사용자가 자연어로 서비스 구현·관리 의도를 전달하면 전문화된 AI 에이전트 팀이 실제 소프트웨어를 만들어내는 **AI 멀티 에이전트 오케스트레이션 플랫폼**입니다.

---

## 지휘자의 역할

xzawedOrchestrator(지휘자)는 사용자와 에이전트 팀 사이의 **통역자이자 조율자**입니다.

- 사용자의 자연어 지시를 받아 **구조화된 작업 명세**로 정제
- xzawedManager(총관리자)를 통해 **전문 에이전트 팀에 작업 위임**
- 에이전트 팀의 진행 상황과 회신을 **자연어로 사용자에게 중계**
- 추가 정보가 필요할 때 **동적 UI 양식**을 생성하여 사용자에게 요청

사용자는 기획·개발·디자인·테스트 등 각 영역의 전문 에이전트와 대화하듯 협업하며, 지휘자는 그 사이에서 의사소통을 조율하고 방향을 잡아줍니다.

---

## 에이전트 계층 구조

```
사용자
  ↕ 자연어 채팅 (Electron 앱)
xzawedOrchestrator  ← 프로젝트 지휘자 (이 프로젝트)
  ↕ Redis Streams (구조화된 작업 지시)
xzawedManager       ← 총관리자 (별도 서비스, 추후 구현)
  ↕ Redis Streams
├── xzawedPlanner   ← 기획자
├── xzawedDeveloper ← 개발자
├── xzawedDesigner  ← 디자이너
├── xzawedTester    ← 테스터
├── xzawedBuilder   ← 빌드관리자
├── xzawedWatcher   ← 이슈관리자
└── xzawedSecurity  ← 보안관리자
```

각 구성 요소는 **독립 서비스**로, 내부에 자체 Claude Orchestrator와 Sub-agents를 포함합니다. 서비스 간 통신은 Redis Streams를 통한 비동기 메시지 교환으로 이루어집니다.

> **Note:** 현재 구현 범위는 xzawedOrchestrator만입니다. xzawedManager 및 하위 에이전트는 추후 별도 프로젝트로 구현됩니다.

## 핵심 가치

| 가치 | 설명 |
|------|------|
| **자연어 지시** | "쇼핑몰 만들어줘"처럼 말하면 알아서 진행 |
| **전문 에이전트 협업** | 기획·개발·디자인·테스트·보안 각 영역 전문 AI가 담당 |
| **실시간 소통** | 메신저 형태로 진행 현황을 실시간 스트리밍으로 확인 |
| **작업 안정성** | 서비스 중단·재시작 시에도 진행 중 작업 유실 없음 |
| **유연한 배포** | 개인 PC, 개인 서버, 팀 서버 모두 동일 앱으로 지원 |

## 사용자 경험

xzawedOrchestrator는 메신저 형태의 Electron 데스크탑 앱으로 사용자와 상호작용합니다.

```
┌─────────────────────────────────────────────────────┐
│  xzawed — 프로젝트 지휘자                            │
├──────────────┬──────────────────────┬────────────────┤
│ 채널         │  채팅               │  동적 UI 패널  │
│              │                      │                │
│  지휘자      │  사용자: 쇼핑몰      │  요구사항      │
│  현황        │  만들어줘            │  양식 작성...  │
│  알림        │                      │                │
│              │  지휘자: 요구사항    │  서비스 유형 ▼ │
│ ─────────── │  양식을 작성해주세요 │  ☑ 상품관리   │
│ 진행 중      │                      │  ☑ 결제       │
│  쇼핑몰      │  사용자: [제출완료]  │  □ 리뷰       │
│   Developer  │                      │                │
│  랜딩        │  지휘자: 개발팀에    │  [제출]        │
│   Designer   │  전달했습니다.       │                │
├──────────────┴──────────────────────┴────────────────┤
│  메시지 입력...                               [전송]  │
└─────────────────────────────────────────────────────┘
```

## 배포 시나리오

### 로컬 단일 사용자
개인 PC에서 앱을 실행하면 서버와 Redis가 자동으로 함께 시작됩니다. 인터넷 연결 없이도 동작하며, 여러 창을 동시에 열어 복수의 서비스를 병렬로 진행할 수 있습니다.

### 개인 원격 서버
Railway, AWS 등 개인 클라우드에 서버를 배포하고 어디서든 Electron 앱으로 접속합니다.

### 팀 공유 서버
팀이 하나의 서버를 공유합니다. 각 팀원의 세션은 격리되어 서로 간섭하지 않습니다.
