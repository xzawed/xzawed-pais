# ADR-001: 서비스 간 통신은 Redis Streams만 사용

**날짜**: 2026-05-15
**상태**: 채택

## 컨텍스트

xzawedPAIS는 9개 독립 서비스로 구성된다. 서비스 간 통신 방법을 결정해야 했다.
후보: (1) 직접 HTTP 호출, (2) gRPC, (3) Redis Streams, (4) 메시지 큐(RabbitMQ 등)

## 결정

모든 서비스 간 통신은 **Redis Streams**만 사용한다. 서비스끼리 직접 import하지 않는다.

스트림 키 규칙: `{출발지}:to-{목적지}:{sessionId}` / consumer group: `{목적지}-consumers`

## 근거

- **느슨한 결합**: 서비스가 서로의 존재를 모른다. 서비스 추가·제거 시 다른 서비스 코드 변경 불필요
- **비동기 내성**: 하위 서비스가 재시작 중이어도 Orchestrator는 계속 동작 (메시지 큐잉)
- **단일 의존성**: Redis 하나로 모든 서비스 간 통신 해결 (운영 복잡도 최소화)
- **재현 가능한 디버깅**: `redis-cli XRANGE` 로 임의 시점의 메시지 흐름 추적 가능

## 결과

- **+** 서비스 경계가 명확하고 독립 배포 가능
- **+** 단일 Redis 인스턴스로 전체 비동기 통신 관찰 가능
- **-** 에이전트 간 직접 통신 불가 (모두 Manager 경유 필요)
- **-** 동기 응답이 필요한 패턴은 polling 또는 별도 채널 필요
- **향후**: ADR-002에서 MCP 기반 에이전트 간 직접 통신 도입 예정
